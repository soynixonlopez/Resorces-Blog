Configuración Inicial y Comandos Básicos

⚙️ Paso 1: Configurar tus Credenciales

Antes de empezar, configura tu identidad en Git:

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

🔍 Paso 2: Verificar Repositorio Remoto

Comprueba si tu repositorio está correctamente conectado:

git remote -v
Si no ves ningún remoto, necesitas configurarlo:
git remote add origin https://github.com/usuario/repositorio.git

📝 Paso 3: Comandos Básicos para Subir Cambios

¡Importante! Siempre verifica en qué rama estás:
git branch

1. Ver el Estado de tus Archivos

git status

2. Agregar Archivos Modificados

git add .

3. Crear un Commit

git commit -m "Descripción clara de tus cambios"

4. Subir los Cambios

git push origin nombre-de-tu-rama

Reemplaza 'nombre-de-tu-rama' con 'main' o el nombre de tu rama actual

Consejo: Si es la primera vez que subes una rama nueva, usa:
git push -u origin nombre-de-tu-rama

Fundamentos del Trabajo en Equipo

🌟 Regla de Oro

Siempre actualiza antes de empezar a trabajar:
Este es el principio más importante del trabajo en equipo con Git.

Paso 1: Actualizar tu Repositorio

git pull origin main

Este comando descarga todos los cambios recientes del repositorio principal.

Paso 2: Crear tu Rama de Trabajo

git checkout -b nombre-de-tu-rama

Si ya tienes una rama, puedes usar: git checkout nombre-de-tu-rama para cambiar a ella.

Mantén tu Código Actualizado

¿Por qué es importante?

Trabajar con código desactualizado es la principal causa de conflictos en Git. Mantener tu código actualizado es como mantener tu teléfono actualizado: evita problemas y te da nuevas características.

📱 Flujo de Trabajo Diario

  1. Antes de empezar a trabajar:
    git pull --rebase origin main

    Esto actualiza tu rama con los últimos cambios de main.

  2. Durante el trabajo:
    git fetch origin

    Verifica si hay cambios nuevos sin aplicarlos.

  3. Antes de subir tus cambios:
    git pull --rebase origin main

    Asegúrate de que tus cambios estén actualizados antes de subirlos.

🔄 Comandos de Actualización

1. Ver Cambios Remotos

git fetch origin

Descarga información sobre cambios sin aplicarlos.

2. Ver Diferencias

git diff origin/main..HEAD

Muestra las diferencias entre tu rama y main.

3. Actualizar con Rebase

git rebase origin/main

Incorpora los cambios de main en tu rama de manera limpia.

Consejos Pro

  • Haz git pull --rebase al menos una vez al día
  • Antes de empezar una nueva tarea, actualiza tu rama
  • Si trabajas en equipo, actualiza antes de subir tus cambios
  • Usa git fetch para ver cambios sin aplicarlos

⚠️ Qué hacer si algo sale mal

Si el rebase falla:

git rebase --abort

Esto te devuelve al estado anterior.

Si hay conflictos:

  1. Resuelve los conflictos en los archivos
  2. Usa git add . para añadir los cambios
  3. Continúa con git rebase --continue

Resolución de Conflictos

⚠️ Casos Comunes de Conflicto

  • Dos personas modifican la misma línea de código
  • Una persona modifica una línea que otra eliminó
  • Cambios en el mismo archivo sin actualizar de main

Paso 1: Identificar el Conflicto

git status

Git te mostrará los archivos con conflictos en rojo.

Paso 2: Entender el Formato del Conflicto

Git marca los conflictos así:

<<<<<<< HEAD
Tu código
=======
Código de tu compañero
>>>>>>> origin/main

Donde:

  • <<<<<<< HEAD: Tu versión del código
  • =======: Separador entre las dos versiones
  • >>>>>>> origin/main: Versión de tu compañero

Paso 3: Ejemplos de Conflictos Comunes

Ejemplo 1: Modificación de la misma línea

<<<<<<< HEAD
const nombre = "Juan";
=======
const nombre = "María";
>>>>>>> origin/main

Solución: Decidir qué nombre usar o combinarlos.

Ejemplo 2: Adición de código en el mismo lugar

<<<<<<< HEAD
function saludar() {
    console.log("Hola");
}
=======
function saludar() {
    alert("Hola");
}
>>>>>>> origin/main

Solución: Elegir una implementación o combinar ambas.

Ejemplo 3: Eliminación vs Modificación

<<<<<<< HEAD
// Código eliminado
=======
// Código modificado
const x = 5;
>>>>>>> origin/main

Solución: Decidir si mantener, eliminar o modificar el código.

Paso 4: Resolver el Conflicto

  1. Abre el archivo con conflictos en tu editor
  2. Busca las marcas de conflicto (<<<<<<<, =======, >>>>>>>)
  3. Edita el archivo para resolver el conflicto
  4. Elimina las marcas de conflicto
  5. Guarda el archivo

Paso 5: Finalizar la Resolución

git add .
git commit -m "Resuelve conflictos de merge"

Consejos Importantes

  • Antes de resolver conflictos, asegúrate de tener una copia de seguridad
  • Comunícate con tu compañero si no estás seguro de qué cambios mantener
  • Si te confundes, puedes usar git rebase --abort para empezar de nuevo
  • Después de resolver conflictos, prueba que todo funcione correctamente

🛠️ Comandos Esenciales

Operaciones fundamentales que debes conocer

Gestión de Ramas

git branch

Ver todas las ramas 📋

git checkout -b nueva-rama

Crear y cambiar a nueva rama 🌱

git checkout nombre-rama

Cambiar a otra rama 🔄

git branch -m nuevo-nombre

Renombrar rama actual ✏️

git branch -d nombre-rama

Eliminar una rama 🗑️

Control de Versiones

git log --oneline

Ver historial de commits 📜

git reset --hard HEAD~1

Deshacer último commit ⏮️

git reset --soft HEAD~1

Deshacer commit manteniendo cambios 🔄

git checkout -- .

Descartar todos los cambios locales 🗑️

git stash

Guardar cambios temporalmente 💾

¡Importante! Los comandos con --hard eliminan cambios permanentemente. ¡Úsalos con precaución! 🚨
